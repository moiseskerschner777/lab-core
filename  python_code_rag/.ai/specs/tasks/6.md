# Phase 6 — Bug Fixes

**Goal:** All issues found during phases 1–5 are resolved. The full pipeline (index → search → collections) runs cleanly on any Python codebase.

**Flow:** Flow 1 — Build→Observe→Adjust

**References:** `design.md` — all sections

---

> **When a task is complete, mark it `[-]`.**

This phase is intentionally empty at planning time.

Add tasks here as bugs are discovered during phases 1–5. Each task follows this format:

```
- [ ] 6.N <short description>

  **Symptom:** what was observed (error message, wrong output, crash)
  **Root cause:** what caused it
  **Fix:** which file and what change

  **Test:**
  - Do: the exact action that previously triggered the bug
  - Expect: correct behavior after the fix
```
