# Phase 4 — Index + Collections Endpoints

**Goal:** `POST http://localhost:8001/index` indexes a mounted codebase into IRIS. `GET /collections` lists all indexed collections. Running index twice on the same path produces the same row count.

**Flow:** Flow 1 — Build→Observe→Adjust

**References:** `design.md` → `routers/index.py`, `routers/collections.py`, `retriever.py`, API Contracts

**Prerequisites:** Phase 3 complete. IRIS and Ollama running.

- [ ] 4.5 Create `app/routers/index.py`. Implement `POST /index`:

  Request model:
  ```python
  class IndexRequest(BaseModel):
      path: str
  ```

  Handler logic:
  1. Validate `Path(request.path).exists()` — raise `HTTPException(400)` if not.
  2. `collection = collection_name(request.path)`
  3. `chunks = chunk_codebase(Path(request.path))`
  4. `vectors = embed([c.text for c in chunks])`
  5. `ensure_table(conn, collection)`
  6. `delete_collection(conn, collection)` — clean re-index
  7. `insert_chunks(conn, collection, chunks, vectors)`
  8. Return `{"collection": collection, "chunks_indexed": len(chunks), "status": "ok"}`

  **Test:**
  - Do: `docker compose run --rm app python -c "from app.routers.index import router; print(router.routes)"`
  - Expect: one POST /index route. No import errors.
