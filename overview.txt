MedBridge — where the project stands today


what we are building:

    an intelligent interoperability platform for the laboratory sector.
    the core idea is a layer that sits between systems and orchestrates communication between them — autonomously, with context, and with risk awareness.

    it is not just an integration. it reasons. it diagnoses. it decides.


the three systems:

    LabCore — System A:

        the main LIS. the source of truth.
        has its own database, its own schema, its own REST API.
        not a FHIR server. speaks its own proprietary format.
        the AI agent reads context from here before making any decision.

    RefLab — System B:

        an external reference laboratory that receives support exams.
        speaks FHIR R4 natively.
        the agent translates LabCore data into FHIR to communicate with it.

    VitaCare — System C:

        an external insurance/covenant system.
        speaks its own proprietary API — not FHIR.
        handles coverage validation and pre-authorization.

the agent flow — cache-first design:

    the key principle: LLM is only invoked when needed.

    when a request arrives:

        check MappingCache first:

            hit  → use saved payload schema directly. no LLM. no cost.
                    if this cached call fails → invalidate cache → agent wakes up.

            miss → agent is invoked:
                    reads KnowledgeBase docs (swagger, PDF, URL)
                    builds payload with LLM
                    sends request

                    SUCCESS → save to MappingCache → future calls bypass LLM entirely
                    ERROR   → diagnose → classify risk → save ErrorReport

    risk classification determines action:

        LOW      → auto-fix and retry
        MEDIUM   → fix + notify human
        CRITICAL → hard block + escalate. agent never auto-fixes this. a life could be lost.

    two permanent tables in IRIS for the agent:

        MappingCache:
            stores what worked. operation + system + payload schema + embedding.
            use_count and last_used_at track staleness.

        ErrorReport:
            stores what failed. always. regardless of outcome.
            full Markdown report: error raw, payload sent, LabCore context, diagnosis, decision.
            embedded as vector so future errors find similar past cases.


the vector store:

    lives natively inside IRIS — no separate database needed.
    two uses:

        KnowledgeBase:
            documentation for System A indexed once.
            swagger files, PDFs, example requests.
            agent queries: "how does this endpoint work?"

        ErrorReport history:
            grows with every failure.
            agent queries: "have I seen something like this before?"
            the more errors happen, the smarter the diagnosis gets.


the mock servers:

    RefLab and VitaCare are FastAPI mocks — no real database.
    they simulate realistic errors via trigger values in the request:

        patient_id: ERR-ACCENT    → 422 name mismatch (accent stripped)
        exam_code:  ERR-FORMAT    → 422 wrong code format (HEM001 vs HEM-001)
        covenant_id: ERR-EXPIRED  → 403 plan not active
        cancel_id: ERR-PROCESSING → 409 sample already in analyzer
        order_id:  ERR-TIMEOUT    → 408 slow response (30s sleep)

        and 45+ other scenarios covering:
            identity mismatches
            clinical data errors
            sample collection errors
            authorization and covenant errors
            timing and workflow errors
            data format and encoding errors
            partial success edge cases
            cancellation-specific failures

    priority scenarios for the contest demo:
        1. exam code format mismatch   → LOW      → agent auto-fixes
        2. covenant expired            → MEDIUM   → agent escalates
        3. duplicate order via history → LOW      → vector search memory
        4. cancel already processing   → CRITICAL → hard block + AuditEvent
        5. exam not covered            → MEDIUM   → agent tries alternative
        6. patient accent mismatch     → CRITICAL → identity risk escalation
